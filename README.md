# 动量轮动模拟盘 momentum-sim

A 股「热门概念板块成分股动量轮动」量化策略模拟盘。每日收盘后评估、次日开盘执行（严格 T+1），
配 Flask + ECharts 实时 Dashboard。

> 仅用于策略演示与教学，非投资建议。

## 策略概览

- **股票池**：沪深 300 + 中证 500 + 中证 1000 成分股并集（约 1800 只）
- **数据源**：baostock 日线（免费、稳定）
- **初始资金**：50 万 · 最多持 10 只 · 等权（每只 10%）
- **费用**：佣金万 3 · 卖出印花税万 5 · 最低 5 元

### 选股流程（每日收盘）

板块分组用 **概念板块**（新浪 gn_*：机器人 / 低空经济 / 算力 / 华为汽车 / BC电池等跨行业短线
题材），比证监会行业分类更贴合 A 股炒作逻辑。一只股票可属多个概念，归到其**最热概念**分组；
概念获取失败时自动退化为申万行业分类（不影响主流程）。

1. **概念热度**：把每只股票计入其所属的每个概念，计算各概念过去 5/20 日平均涨幅，取 **Top 30%** 热门概念；
2. **个股归组**：一只股票可属多个热门概念，取其中**最热概念**作为该股分组；
3. **个股动量评分**（仅热门概念内）：

   | 因子 | 权重 | 计算 |
   |------|------|------|
   | MOM_5 | 20% | 过去 5 日涨幅 |
   | MOM_20 | 30% | 过去 20 日涨幅（去掉最近 1 日） |
   | 量价共振 | 25% | 近 5 日量能扩张 × 上涨（放量上涨） |
   | 相对板块强度 | 25% | 个股 MOM_20 − 所在概念平均 MOM_20 |

   各因子在概念内排名归一化（0-1）后加权求综合分；
4. **选股**：全市场综合分 Top 20 候选，过滤停牌 / ST / 成交额 < 1 亿 / 跌停，同一概念最多 3 只，同股去重；
5. **卖出**：所属概念跌出热门 Top30% · 个股排名跌出全市场 Top30% · 回撤 > -8% · 持有 > 15 自然日；
6. **执行**：T 日收盘选股 → T+1 开盘价买入 / 卖出（严格 T+1，不做盘中）。

## 项目结构

```
momentum-sim/
├── data_fetcher.py    # baostock 日线面板 + 新浪概念板块映射(行业分类兜底)
├── strategy.py        # 概念热度 + 个股动量评分 + 选股 + 卖出判定
├── engine.py          # 撮合执行（T+1 开盘买卖）
├── database.py        # SQLite 持久化
├── scheduler.py       # 每日收盘后结算
├── app.py / wsgi.py   # Flask Dashboard（端口 8889）
├── templates/index.html
├── data/              # panel.db（日线面板） + trading.db（账户/交易）
└── requirements.txt
```

## 运行

```bash
pip install -r requirements.txt

# 1) 构建日线面板（沪深300+中证500+中证1000，约 20 分钟）
python3 data_fetcher.py panel 2026-04-01 2026-06-22

# 2) 启动 Dashboard
python3 wsgi.py            # http://localhost:8889

# 3) 启动收盘结算调度器
python3 scheduler.py
```

或用 pm2：`pm2 start ecosystem.config.js`

## 数据源说明

baostock 仅提供日线，本策略不依赖盘中数据——所有决策基于 T 日完整收盘日线，
次日开盘价执行，因此**无未来函数**：候选池在 T 日生成，T+1 开盘成交。
